3.1.2 连续分配管理方式（重要）

1.单一连续分配

用户区内仅有一道用户程序，即用户程序独占整个用户区。

优点：

  • 简单，无外部碎片（因为所有的用户区都分配给了一个用户程序）

  • 无序进行内存保护，因为只有一道程序

缺点

  • 内部碎片

  • 内存利用率低

2.固定分区分配

多道程序的存储管理方式，内存空间划分为固定大小的分区。

注意，固定不代表分区大小一定相等！

  • 分区大小相等：程序太小会造成浪费；太大无法存入。

  • 分区大小不等：具有灵活性。

3.动态分区分配

不会像固定内存区域一样实现划分内存分区，而是在进程装入内存的时候根据进程的大小动态简历分区。

动态分区分配使用什么数据结构？

  • 空闲分区表

  • 空闲分区链

动态分区如何进行分配与回收？

分成4种情况（本质上就是合并相邻的空闲区）

  1. 回收区之后有相邻的空闲分区

    • 描述：回收的内存分区后面紧接着一个空闲分区。

    • 处理方式：将回收区与后面的空闲分区合并，形成一个更大的空闲分区。

    • 优点：减少内存碎片，提高内存利用率。

    • 示例：

      • 回收区地址：1000-2000

      • 空闲分区地址：2000-3000

      • 合并后空闲分区：1000-3000

  2. 回收区之前有相邻的空闲分区

    • 描述：回收的内存分区前面紧接着一个空闲分区。

    • 处理方式：将回收区与前面的空闲分区合并，形成一个更大的空闲分区。

    • 优点：减少内存碎片，提高内存利用率。

    • 示例：

      • 空闲分区地址：500-1000

      • 回收区地址：1000-2000

      • 合并后空闲分区：500-2000

  3. 回收区前、后都有相邻的空闲分区

    • 描述：回收的内存分区前后都有相邻的空闲分区。

    • 处理方式：将回收区与前后两个空闲分区合并，形成一个更大的空闲分区。

    • 优点：最大限度地减少内存碎片，提高内存利用率。

    • 示例：

      • 空闲分区地址：500-1000

      • 回收区地址：1000-2000

      • 空闲分区地址：2000-3000

      • 合并后空闲分区：500-3000

  4. 回收区前、后都没有相邻的空闲分区

    • 描述：回收的内存分区前后都没有相邻的空闲分区。

    • 处理方式：将回收区单独标记为空闲分区，不进行合并。

    • 优点：简单直接，适用于孤立的内存分区。

    • 示例：

      • 回收区地址：1000-2000

      • 空闲分区地址：500-1000（不相邻）

      • 空闲分区地址：2000-3000（不相邻）

      • 回收后新增空闲分区：1000-2000

紧凑技术

没有内部碎片，但是会有外部碎片。外部碎片可以通过紧凑技术。

紧凑技术是指将内存中已分配的分区（占用区）向一端移动，同时将空闲分区合并，从而减少内存碎片，形成一个连续的大空闲分区。这一过程也称为内存碎片整理。

动态分区分配算法（重要）

  1. 首次适应算法（First Fit）​

    • 原理：从内存的起始位置开始搜索，找到第一个能够满足请求大小的空闲分区，并分配给进程。

    • 特点：

      • 简单高效，搜索速度快。

      • 倾向于在内存的低地址部分分配，可能导致低地址部分碎片化。

    • 优点：实现简单，分配速度快。

    • 缺点：可能产生外部碎片，内存利用率较低。

    • 适用场景：对分配速度要求较高的场景。

  2. 最佳适应算法（Best Fit）​

    • 原理：搜索整个空闲分区链表，找到能够满足请求大小的最小空闲分区，并分配给进程。

    • 特点：

      • 尽量利用小分区，减少大分区的浪费。

      • 可能导致大量小碎片，增加外部碎片。

    • 优点：内存利用率较高，适合小内存请求。

    • 缺点：搜索速度较慢，可能产生大量小碎片。

    • 适用场景：内存资源有限且请求大小差异较大的场景。

  3. 最坏适应算法（Worst Fit）​

    • 原理：搜索整个空闲分区链表，找到最大的空闲分区，并分配给进程。

    • 特点：

      • 尽量利用大分区，减少小分区的产生。

      • 可能导致大分区被快速消耗，无法满足后续大内存请求。

    • 优点：减少小碎片的产生。

    • 缺点：大分区被快速消耗，可能无法满足大内存请求。

    • 适用场景：内存请求大小较为均匀的场景。

  4. 邻近适应算法（Next Fit）​

    • 原理：从上次分配结束的位置开始搜索，找到第一个能够满足请求大小的空闲分区，并分配给进程。

    • 特点：

      • 分配较为均匀，避免集中在低地址部分。

      • 可能导致高地址部分碎片化。

    • 优点：分配速度较快（不用再给空闲分区链重新排序），避免低地址部分碎片化。

    • 缺点：可能产生外部碎片，内存利用率较低。

    • 适用场景：对分配速度要求较高且内存请求分布均匀的场景。

3.1.3 基本分页存储管理（重要，注意文字）

分页存储的几个基本概念

  • 内存地址——页框、页帧、物理块。

  • 逻辑地址——页、页面。

基本地址变化机构

  • 页表寄存器（PTR）存放页表在内存中的起始地址F和页表长度M，在单CPU系统中只设置一个页表寄存器。

  • 平时的页表起始地址和页表长度存放在本进程的PCB中。

3.1.4 基本分段存储管理（重要）

分段：按照程序自身的逻辑关系划分成若干段，每段从0开始编址。

内存分配规则：每个段内部连续，但是段之间可以不相邻。

逻辑地址组成

逻辑地址由段号和段内地址（段内偏移量）组成

  • 段号位数决定可以分多少个段

  • 段内地址位数决定每个段的最大长度是多少

段表

  • 相比于页表多了一列表示段长，这是因为每个段长可能是不同的，但是分页中每个页的长度都是相同的。

  • “段表项”长度是相同的，所以同样段表项中的段号可以隐含，不占存储空间。

本章问题

编译、链接、装入的作用分别是什么？

  • 编译：由代码文件转化为目标模块（高级语言转化为机器语言）。

  • 链接：由目标模块生成装入模块，链接后形成完整的逻辑地址。

  • 装入：装入模块进入内存，装入后形成物理地址。

分段，分页管理的对比？（重要）

总结一下：

  • 页是信息的物理单位，对用户不可见；段是信息的逻辑单位，对用户可见。

  • 分页的用户进程的地址空间是一维的，分段的地址空间是二维的。分页机制将逻辑地址空间划分为大小固定且连续的页（如4KB），用户进程的逻辑地址仅需通过一个线性地址即可表达。

    • 例如，当用户给出逻辑地址5000时，系统自动根据页大小（如4KB）计算出页号（第2页）和页内偏移（1000），无需用户显式指定页号。

    • 分段管理根据程序的逻辑功能（如代码段、数据段、栈段）划分内存，每个段独立编址且长度可变。用户需显式指定段名（或段号）和段内偏移量，例如汇编指令MOV [DS:0x100]需同时指明段（DS）和偏移地址（0x100），形成二维地址结构。

  • 分段比分页更容易实现信息保护（上图2、3）
