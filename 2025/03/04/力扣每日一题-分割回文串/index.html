132.分割回文串2

和分割回文串1有很大关联。

1278.分割回文串3

上一个问题是两个动态规划相结合，这题也是。要先用编辑距离的思想求出对于某一区间的字符串变成回文串的最小编辑次数，然后就得到递推公式就方便了——dp[i][j] = Math.min(dp[i][j], dp[t][j - 1] + cost[t + 1][i]);

问题描述

给定一个由小写字母组成的字符串 s 和一个整数 k，要求：

  1. 可以修改 s 中的部分字符为其他小写字母。
  2. 将 s 分割成 k 个非空且不相交的子串，且每个子串都是回文串。
  3. 返回满足上述条件所需修改的最少字符数。

思路

这个问题可以拆解为两个部分：

  1. 计算将任意子串变成回文串的最小修改次数。

  2. 在字符串中寻找 k 个分割点，使得总修改次数最小。

  3. 计算将子串变成回文串的最小修改次数

定义一个二维数组 cost[i][j]，表示将子串 s[i...j] 变成回文串所需的最小修改次数。计算 cost[i][j] 的方法：

  • 如果 s[i] == s[j]，则 cost[i][j] = cost[i+1][j-1]（不需要修改）。
  • 如果 s[i] != s[j]，则 cost[i][j] = cost[i+1][j-1] + 1（需要修改其中一个字符）。
  1. 动态规划寻找最佳分割点

定义一个二维数组 dp[i][j]，表示将字符串 s[0...i] 分割成 j 个回文子串所需的最小修改次数。状态转移方程：

  • 如果 j == 1，则 dp[i][1] = cost[0][i]（整个字符串作为一个回文子串）。
  • 如果 j > 1，则遍历所有可能的分割点 t，并更新 dp[i][j] = min(dp[i][j], dp[t][j-1] + cost[t+1][i])。
