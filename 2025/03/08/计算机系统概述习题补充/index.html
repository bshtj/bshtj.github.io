第一章 计算机系统概述

1.1 操作系统的基本概念

操作系统是对什么管理的软件？

操作系统是对计算机资源 管理的软件

  • 计算机资源包括软件和硬件

操作系统提供给编程人员的接口是？

系统调用。

这里复习一下：操作系统作为用户和计算机硬件系统之间的接口，一共提供两种接口

  • 命令接口（命令行）

    • 联机命令接口（交互式命令接口）如命令行

    • 脱机命令接口（批处理命令接口）如windows中光电bat

  • 程序接口（GUI）：由一组系统调用（广义指令组成。GUI就是通过调用程序接口实现的）

系统调用和库函数关联？

库函数位于用户控件；系统调用位于内核空间，库函数使用系统调用。

有的库函数会提供系统调用。如mmap之类的。

1.2 操作系统发展历程

1.2.1 手工操作阶段（无操作系统）

计算机所有的工作都需要人工干预，如装入、运行、输出等。

缺点：

  • 用户独占全机，不会出现资源被其他用户占用而等待（因为独占，只能运行一个），但是资源利用率低。

  • CPU等待手工操作，CPU利用不充分

1.2.2 批处理阶段（操作系统出现）

  1. 单道批处理

    将一批作业机脱机的方式输入磁带，并使用系统中的Mointor控制使得这批作业可以一个接一个地连续处理。

    特点：

    • 自动性。mointor监督逐个运行

    • 顺序性。fcfs

    • 单道性。内存中只有一道程序运行。

    由弹道性可知单道批处理会造成“高速的cpu和低速的IO”。因此为了提升IO的效率，设计了躲到批处理。

所有的作业首先在外存上排成一个队列，作业调度在按照一定的算法从后备队列中选择若干作业调入程序。

在管理程序安排下交替运行。

特点：宏观上并行、微观上串行。

1.2.3 分时操作系统

分时操作系统是一种允许多个用户通过终端同时使用计算机资源的操作系统。它的核心特点是通过时间片轮转的方式，将处理器时间分配给多个用户或任务，从而实现多用户共享系统资源的目标。

特点：

虽然分时操作系统较好的解决了人机交互问题，但是有些时候要求做出反映的时间要比时间片更短。——实时操作系统。

1.2.4 实时操作系统

实时操作系统的主要追求目标是什么？

实时操作系统的主要追求目标是：

  • 实时性：确保任务在规定的时间内完成。

  • 可靠性：保证系统在关键任务中的稳定性和安全性。

资源利用率并不是实时操作系统的主要目标，因为实时操作系统可能会通过任务抢占和资源预留来保证实时性和可靠性，这可能导致资源利用率的降低。

作业的几种形式？

作业的几种形式和操作系统的种类对应。

多道程序技术的前提是系统具有？

中断功能。

多道城西技术要求进程间能实现并发，并发需要中断功能的支持。

下列关于多道程序系统的叙述中，判断正确与否。

  1. 支持进程的并发运行（√）

  2. 不必支持虚拟存储技术（√）

  3. 需要实现对共享资源的管理（√）

  4. 进程数越多CPU利用率越高（X）（因为进程越多，竞争越激烈，容易发生死锁。）

其实这道题是用来判断操作系统的基本特点：并发、共享、虚拟、异步。（并发和共享是最基础的特征，两者相互依存）因此1和3是必须的，不用虚拟存储也可以实现，那么如何实现呢？

不用虚拟存储技术如何实现多道程序系统？（见第三章）

虚拟存储技术（如分页、分段）是为了解决内存管理问题（内存碎片、内存利用绿豆）而引入的，但多道程序系统可以通过以下方式实现内存管理，而无需虚拟存储技术：

  1. 固定分区：

    • 将内存划分为固定大小的分区，每个作业分配到适合的分区中。

    • 优点是实现简单，缺点是可能造成内存浪费（内部碎片）。

  2. 动态分区：

    • 根据作业的大小动态分配内存。

    • 优点是内存利用率高，缺点是可能产生外部碎片，需要定期进行内存整理（压缩）。

  3. 覆盖技术：

    • 将程序分为多个模块，只有当前需要的模块加载到内存中，其他模块存储在磁盘上，需要时再加载。

    • 适用于内存有限的系统。

  4. 交换技术：

    • 将暂时不运行的作业从内存换出到磁盘，腾出内存空间给其他作业。

    • 需要时再将作业从磁盘换入内存。

中断与异常

发生中断的整体流程

  1. 中断触发（硬件操作）

  • 事件发生：外设（如键盘、硬盘、网卡等）完成操作或需要CPU介入时，向CPU发送中断信号。此信号通过中断控制器（如APIC或8259A PIC）传送。

  • 硬件检测：CPU在每条指令执行完毕后，检查是否有未被屏蔽的中断请求，查看中断引脚的电平或中断控制器的状态。

  • 中断响应（硬件操作）

  • 中断优先级判断：如果有多个中断请求，硬件中的中断控制器会根据中断的优先级选择最高优先级的中断。

  • 保存关键现场（硬件自动完成）：

    • 保存程序计数器（PC）的值，也就是下一条待执行指令的地址。

    • 保存状态寄存器（如Flags/PSW）的值，记录CPU的状态（如中断使能标志、溢出标志等）。

  • 切换至内核态：CPU自动切换到内核模式（特权模式），并关闭中断（通过清除中断使能标志，以防止嵌套中断）。

  • 跳转到中断处理程序（硬件操作）

  • 查询中断向量表：CPU根据中断类型（如IRQ编号），从中断向量表（IVT）或中断描述符表（IDT）中读取对应的中断服务例程（ISR）入口地址。

  • 跳转执行：CPU将控制权转移到ISR的入口地址，开始执行操作系统的中断处理程序。

  • 保存完整上下文（操作系统操作）

  • 操作系统保存寄存器：在ISR中，操作系统需要保存当前进程的完整上下文（如通用寄存器、段寄存器等），以便中断处理完成后能够恢复程序执行。

    • 例如，在x86架构中，操作系统可能使用pusha指令保存所有通用寄存器。

  • 区分内核态和用户态：如果中断发生时CPU处于用户态，操作系统需要切换栈（从用户栈切换到内核栈）。

  • 处理中断（操作系统操作）

  • 调用设备驱动程序：操作系统根据中断类型，调用相应的设备驱动程序进行处理。

    • 例如，当网卡收到数据包后触发中断，操作系统会调用网卡驱动程序来读取数据，并将其放入接收队列中。

  • 执行必要操作：操作系统执行相关操作，如读取设备数据、发送响应，或更新内核状态，通知其他进程中断处理已完成。

  • 中断结束通知（硬件+操作系统协作）

  • 发送EOI（End of Interrupt）：操作系统向中断控制器发送EOI信号，通知中断已处理完成。

    • 在x86架构中，使用out指令向8259A PIC发送EOI。

  • 重新启用中断：操作系统通过设置状态寄存器（如x86中的sti指令）重新启用中断，允许接收新的中断信号。

  • 恢复上下文（操作系统操作）

  • 恢复寄存器：操作系统从栈中恢复之前保存的寄存器值（如通用寄存器、状态寄存器等）。

  • 切换回用户态：如果中断发生时CPU处于用户态，操作系统会切换回用户栈，并恢复用户态的权限。

  • 返回原程序（硬件操作）

  • 执行中断返回指令：操作系统通过iret（x86架构）或类似指令，触发硬件操作，恢复程序计数器（PC）和状态寄存器（Flags），返回中断前的程序地址和状态。

  • 继续执行原程序：CPU从被中断的指令或下一条指令开始继续执行原程序。

发生异常中断的整体流程

异常（Exception）处理在操作系统中是一项重要机制，确保当程序出现异常时，系统能够有效地响应并进行处理。异常的处理流程与中断（Interrupt）类似，但由于异常是由程序内部执行中的同步事件触发，因此它需要特定的步骤来处理中断和恢复程序。下面是异常处理的整体流程，并详细区分硬件操作与操作系统操作：

  1. 故障（Fault）：

    • 这类异常是可修复的，比如页缺失异常。发生故障后，操作系统会处理并修复问题，然后重新执行触发异常的指令。

  2. 自陷（访管）（Trap）：

    • 这类异常通常用于调试或系统调用。处理后，程序会继续执行下一条指令。

  1. 终止（Abort）：

    • 这类异常表示严重错误，无法恢复，通常会直接终止进程。例如，硬件故障等。

1. 异常触发（硬件操作）

  • 事件发生：CPU在执行指令时检测到异常条件。例如：

    • 除零错误：除法指令的除数为零。

    • 页缺失：访问的虚拟地址没有映射到物理内存。

    • 非法指令：执行了未定义或非法的指令。

  • 异常是同步的，即由当前正在执行的指令直接引发。

2. 硬件响应（硬件操作）

  • 保存关键现场：

    • 程序计数器（PC）：硬件会自动保存触发异常的指令地址，或者是下一条指令的地址，取决于异常类型。

    • 状态寄存器（Flags/PSW）：保存当前的CPU状态，如中断使能标志和特权级别。

  • 切换至内核态：CPU会自动切换到内核模式，并禁用中断以防止异常嵌套。

  • 生成错误码（可选）：对于部分异常（如页缺失），会生成错误码并保存到栈中，供操作系统进一步分析。

3. 跳转到异常处理程序（硬件操作）

  • 查询异常向量表：CPU根据异常类型（如除零、页缺失等）从异常向量表（如中断描述符表IDT）中查找对应的处理程序入口。

    • 例如，在x86架构中：

      • 除零异常对应向量号为0。

      • 页缺失异常对应向量号为14。

  • 跳转执行：CPU将控制权转交给操作系统的异常处理程序，如 divide_error 或 page_fault_handler。

4. 保存完整上下文（操作系统操作）

  • 手动保存寄存器：操作系统会保存当前进程的通用寄存器、段寄存器等上下文信息，以便后续恢复。

  • 区分用户态和内核态：

    • 如果异常发生在用户态，操作系统需要切换栈（从用户栈切换到内核栈）。

    • 如果异常发生在内核态，操作系统可能直接终止进程（如内核代码触发不可恢复的错误）。

5. 处理异常（操作系统操作）

  • 分析异常原因：操作系统会读取错误码（如果有的话）并分析导致异常的指令地址。

    • 例如，在页缺失异常中，操作系统需要检查访问的虚拟地址和相应的权限（如读/写/执行）。

  • 分类型处理：

    • 故障（Fault）：

      • 页缺失：操作系统会分配物理内存或从磁盘加载数据，修复页表后重新执行触发异常的指令。

      • 除零错误：操作系统可能向进程发送信号（如 SIGFPE），并终止进程。

    • 陷阱（Trap）：

      • 系统调用：系统调用通常通过 int 0x80 或 syscall 指令触发，处理后继续执行下一条指令。

    • 终止（Abort）：

      • 操作系统会直接终止进程，记录错误日志，并可能触发内核崩溃（如内核遇到严重错误时）。

6. 恢复或终止（操作系统操作）

  • 可恢复异常（如页缺失）：

    • 操作系统恢复寄存器，并通过 iret 指令返回，继续执行触发异常的指令。

  • 不可恢复异常（如非法指令）：

    • 操作系统向进程发送信号（如 SIGSEGV）终止进程，并释放相关资源。

    • 如果异常发生在内核态，可能触发系统崩溃（例如 Kernel Panic）。

7. 返回原程序或调度新进程（硬件与操作系统协作）

  • 执行返回指令：操作系统通过 iret 指令恢复硬件现场，从栈中弹出保存的程序计数器（PC）和状态寄存器。CPU恢复到用户态（如果需要）并重新开启中断。

  • 继续执行：

    • 如果异常已经修复（如页缺失），CPU会重新执行触发异常的指令。

    • 如果进程被终止，操作系统会调度其他进程运行，确保系统稳定运行。

系统调用与一般调用

补充：系统调用需要将CPU中的PSW和PC端值压栈，而一般调用不需要。

处理外部中断的时候，判断以下哪些由硬件完成，哪些由操作系统完成

PC的内容——硬件

TLB的内容——硬件

cache中的内容——硬件

置PSW状态字——硬件

保存通用寄存器中的内容——操作系统

提供中断服务——操作系统

保中断屏蔽字——操作系统

初始化中断向量表（操作系统初始化的时候就完成了）——操作系统

硬件与操作系统的职责划分

1. 硬件管理的资源

指令执行：CPU执行指令、计算、内存访问。

中断处理：

检测中断信号（如时钟中断、IO完成中断）。

自动跳转到预设的中断处理程序入口（如x86的IDT表）。

基本寄存器保存：

在中断/异常发生时，自动保存PC和PSW到内核栈。

内存管理单元（MMU）：

虚拟地址到物理地址的转换（页表遍历）。

TLB管理（部分架构由硬件自动填充）。

缓存一致性：通过协议（如MESI）维护多核Cache一致性。

2. 操作系统管理的资源

进程上下文：

保存/恢复通用寄存器、堆栈指针、进程状态。

管理进程控制块（PCB）。

内存管理：

分配/回收物理内存，维护页表结构。

处理缺页异常，触发TLB刷新。

设备管理：

提供设备驱动，控制IO操作（如磁盘读写）。

处理DMA与内存的一致性（刷新Cache）。

调度策略：

决定进程切换的时机和目标（如时间片轮转、优先级调度）。

3. 协同工作的典型案例

系统调用（Syscall）：

硬件：执行syscall指令，切换到内核态，保存用户态PC和PSW。

操作系统：保存用户态通用寄存器，执行系统调用处理函数，最后恢复寄存器并返回用户态。

缺页异常：

硬件：触发缺页异常，保存当前PC和PSW。

操作系统：分配物理页，更新页表，刷新TLB，重新执行指令。

硬件与操作系统的职责边界

  • 硬件：执行指令、处理中断、基础地址转换、维护缓存一致性。
  • 操作系统：管理进程上下文、内存分配、设备IO、调度策略。

  • 扩展性与灵活性

  • 可靠性与安全性

  • 可移植性

  • 分布式计算

计算机开机流程

1. CPU通电，执行第一条指令

  • **地址FFFF0H**：CPU通电后，会从固定地址FFFF0H（实模式下的复位向量）获取第一条指令。
  • JMP指令：该地址通常存储一条JMP指令，跳转到BIOS的入口点（通常在F0000H附近）。

2. BIOS初始化

  • 中断向量表和服务程序：BIOS在内存的低地址空间（如00000H到003FFH）构建中断向量表（IVT），并加载相应的中断服务程序。
  • 硬件初始化：BIOS初始化硬件设备（如键盘、显示器、内存、硬盘等）。
  • POST（通电自检）：BIOS执行POST，检测硬件是否有故障。如果发现问题，会通过蜂鸣声或屏幕显示错误代码。

3. 查找启动设备

  • 启动顺序：BIOS按照配置的启动顺序（如硬盘、光驱、USB、网络）查找可启动设备。

  • MBR的加载：

    • 如果选择从硬盘启动，BIOS会读取硬盘的第一个扇区（512字节），即MBR（主引导记录）。
    • MBR中包含：
      • 引导代码（446字节）：负责加载操作系统内核或下一级引导程序。
      • 分区表（64字节）：描述硬盘的分区信息。
      • 结束标志（2字节，0x55AA）：用于验证MBR的有效性。

4. MBR的执行

  • 引导代码：MBR中的引导代码被执行，其任务是查找活动分区（标记为可启动的分区）。

  • PBR的加载：

    • 找到活动分区后，MBR会读取该分区的第一个扇区，即PBR（分区引导记录）。
    • PBR中包含：
      • 引导代码：负责加载操作系统的引导程序（如Windows的NTLDR或Linux的GRUB）。
      • 文件系统信息：用于定位操作系统内核。

5. 引导加载程序（Bootloader）

  • GRUB（Linux）或NTLDR（Windows）：PBR中的引导代码加载更复杂的引导加载程序。
  • 加载操作系统内核：引导加载程序读取配置文件，定位操作系统内核文件（如vmlinuz或ntoskrnl.exe），并将其加载到内存中。

6. 操作系统内核初始化

  • 内核启动：操作系统内核接管控制权，初始化硬件、加载驱动程序、挂载文件系统。
  • 用户空间初始化：启动init进程（Linux）或winlogon进程（Windows），进入用户登录界面。
