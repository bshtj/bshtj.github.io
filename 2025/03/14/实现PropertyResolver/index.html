实现PropertyResolver

https://liaoxuefeng.com/books/summerframework/ioc/property-resolver/index.html

Spring的注入分为@Autowired和@Value两种。对于@Autowired，涉及到Bean的依赖，而对于@Value，则仅仅是将对应的配置注入，不涉及Bean的依赖，相对比较简单。为了注入配置，我们用PropertyResolver保存所有配置项，对外提供查询功能。

本节我们来实现PropertyResolver，它支持3种查询方式：

  1. 按配置的key查询，例如：getProperty("app.title");
  2. 以${abc.xyz}形式的查询，例如，getProperty("${app.title}")，常用于@Value("${app.title}")注入；
  3. 带默认值的，以${abc.xyz:defaultValue}形式的查询，例如，getProperty("${app.title:Summer}")，常用于@Value("${app.title:Summer}")注入。

Java本身提供了按key-value查询的Properties，传入Properties，内部按key-value存储。

关于Java提供的Properties

Properties 是 Java 中的一个类，位于 java.util 包中，用于管理键值对（key-value pairs）形式的配置数据。它继承自 Hashtable<Object, Object>，但通常用于处理字符串类型的键和值。Properties 类常用于读取和写入配置文件（如 .properties 文件）。

properties特点

  1. 配置文件支持：

可以从 .properties 文件加载数据，或将数据保存到 .properties 文件。Properties 是 Java 中专门用于处理键值对配置数据的类，它继承自 Hashtable，但在设计上有一些特定的优势，使其在处理配置文件时比 Hashtable 或其他键值对集合（如 HashMap）更合适。以下是 Properties 的优势和使用场景：

  1. 专为配置设计

Properties 是为管理配置数据而设计的，它的 API 更加简洁和直观，适合处理字符串类型的键值对。例如：

  • setProperty(String key, String value)：设置键值对。
  • getProperty(String key)：获取键对应的值。
  • getProperty(String key, String defaultValue)：获取值，如果键不存在，返回默认值。

这些方法专门为配置场景设计，比 Hashtable 的通用方法（如 put() 和 get()）更符合配置管理的需求。

  1. 配置文件支持

Properties 提供了直接读写配置文件的功能，这是它最大的优势之一：

  • 加载配置文件：通过 load(InputStream in) 方法，可以从 .properties 文件或输入流中加载配置。
  • 保存配置文件：通过 store(OutputStream out, String comments) 方法，可以将配置保存到文件或输出流中。

这种功能是 Hashtable 和其他键值对集合所不具备的。
