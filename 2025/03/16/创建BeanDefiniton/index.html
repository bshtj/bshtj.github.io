主要实现

扫描获取所有Bean的Class类型

注释 // 扫描获取所有Bean的Class类型 的意思是：通过扫描 configClass 中 @ComponentScan 注解指定的包路径，获取所有标注了 Spring 注解（如 @Component、@Service、@Repository、@Controller 等）的类名。这些类名通常是类的全限定名（fully qualified name），例如 com.example.MyService。

  1. 扫描包路径：
    1. scanForClassNames(configClass) 方法会查找 configClass 上的 @ComponentScan 注解。
    2. 如果 @ComponentScan 注解指定了包路径，则扫描这些包路径下的所有类。
    3. 如果未指定包路径，则默认扫描 configClass 所在的包路径。
  2. 获取类名：
    1. 扫描到的类名会被收集到一个 Set<String> 中。
    2. 这些类名是类的全限定名，例如 com.example.MyService。
  3. Bean 的 Class 类型：
    1. 这些类名对应的类通常标注了 Spring 的注解（如 @Component、@Service 等），因此它们是 Spring 容器需要管理的 Bean。

bean的申明类型

这里我们要特别注意一点，就是Bean的声明类型。对于@Component定义的Bean，它的声明类型就是其Class本身。然而，对于用@Bean工厂方法创建的Bean，它的声明类型与实际类型不一定是同一类型。

上述createDataSource()定义的Bean，声明类型是DataSource，实际类型却是某个子类，例如HikariDataSource，因此要特别注意，我们在BeanDefinition中，存储的beanClass是声明类型，实际类型不必存储，因为可以通过instance.getClass()获得。

所以由此可以引出以下两种获得bean的形式。

获取bean name

对应语句

name = (String) anno.annotationType().getMethod("value").invoke(anno);

使用示例（通过反射获取value值，也就是component中的name，也就是bean的名称）

  1. clazz.getAnnotations():
    1. 这行代码用于获取指定类（clazz）上声明的所有注解。clazz 是一个类的 Class 对象，getAnnotations() 方法返回该类上所有注解的数组，返回的是一个 Annotation[] 数组。
  2. for (Annotation anno : clazz.getAnnotations()):
    1. 通过遍历所有的注解实例，逐个处理每个注解。
  3. anno.annotationType():
    1. 通过调用 anno.annotationType() 获取注解的类型（Class<? extends Annotation>）。这一步相当于获取该注解的 Class 对象。
  4. getMethod("value"):
    1. 通过调用注解的 Class 对象的 getMethod("value") 方法，反射获取注解中的名为 "value" 的方法。这里假设注解中有一个名为 value 的方法。
  5. invoke(anno):
    1. 通过反射调用 value 方法，从注解实例 anno 中获取 value 属性的值。返回值是 Object 类型，需要通过强制转换转换为 String 类型。
  6. (String):
    1. 将返回的 Object 类型的值强制转换为 String 类型。
  7. name = (String) anno.annotationType().getMethod("value").invoke(anno);:
    1. 这一行代码完成了通过反射调用注解的 value 方法，获取注解属性值，并将其赋值给变量 name。
