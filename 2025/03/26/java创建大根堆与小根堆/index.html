二叉树结点关系公式

1.二叉树节点关系公式

  1. 父节点索引计算
    • 已知子节点索引为 i，其父节点索引为 **(i-1)/2**（向下取整）。
    • 示例：子节点索引为 5，父节点为 (5-1)/2=2。

  2. 子节点索引计算
    • 已知父节点索引为 i，其左子节点索引为 **2i+1**，右子节点索引为 **2i+2**。
    • 示例：父节点索引为 2，左子节点为 2*2+1=5，右子节点为 2*2+2=6。

  3. 最后一个非叶子节点索引
    • 对含 n 个节点的完全二叉树，最后一个非叶子节点索引为 **n/2-1**（适用于数组存储）。
    • 示例：数组长度为 10，则最后一个非叶子节点索引为 10/2-1=4。

2.完全二叉树特性与存储

  1. 顺序存储规则
    • 完全二叉树的数组存储中，节点按层序从左到右依次存放，索引关系严格满足上述公式。
    • 应用场景：堆（大根堆/小根堆）、优先队列的实现。

  2. 完全二叉树判定条件
    • 除最后一层外，其他层节点数达到最大值，且最后一层节点从左向右连续排列。

  3. 节点数量与深度关系
    • 深度为 h 的完全二叉树最多有 2^h-1 个节点，最少有 2^(h-1) 个节点。

3.其他关键关系

  1. 层序与索引关系
    • 根节点位于第 1 层（部分定义中为第 0 层），子节点层数比父节点大 1。
    • 层序为 L 的节点最大索引为 **2^L-2**（若根为第 1 层）。

  2. 叶子节点判定
    • 若节点索引 i 满足 **i >= n/2**（n 为总节点数），则该节点为叶子节点。

  3. 树的度与高度
    • 树的度：所有节点中最大的子节点数（二叉树度为 2）。
    • 树的高度：从根到最远叶子节点的路径长度。

4.应用场景中的特殊计算

  1. 堆结构调整
    • 构建堆时，从最后一个非叶子节点（n/2-1）开始依次向上调整。
    • 示例：堆排序中，通过 sink() 或 swim() 操作维护堆性质。

  2. 二叉搜索树（BST）
    • 左子树所有节点值小于根，右子树所有节点值大于根。
    • 中序遍历结果为有序序列。

5.总结对比表

关系类型 公式/规则 适用场景 来源
父节点索引 (i-1)/2 堆、完全二叉树数组存储
子节点索引 左：2i+1，右：2i+2 完全二叉树、堆调整
最后一个非叶子节点 n/2-1 堆初始化
完全二叉树层序 层 L 最大索引 2^L-2（根为第 1 层） 树深度计算

通过以上规则，可以高效实现二叉树的遍历、堆操作及存储结构转换。具体代码实现时需注意数组索引从 0 还是 1 开始（如堆的某些实现会从索引 1 开始以简化计算）。

3. 关键点解析

  1. 插入操作 (insert)
    • 新元素放到末尾，通过 上浮 调整位置。
    • 时间复杂度：O(log n)。

  2. 提取最大值 (extractMax)
    • 取出堆顶后，将末尾元素移到堆顶，通过 下沉 (heapify) 调整。
    • 时间复杂度：O(log n)。

  3. 构建堆 (buildHeap)
    • 从 最后一个非叶子节点 开始向前遍历，对每个节点执行 heapify。
    • 时间复杂度：O(n)（比逐个插入的 O(n log n) 更高效）。

  4. 堆化 (heapify)
    • 递归或迭代地比较当前节点与子节点，确保父节点始终最大。
