4.4 路由算法与路由协议

发表于2025-04-08|更新于2025-06-20|笔记408
|浏览量:

画圈的这两种是用在自治系统AS内部的协议

  1. RIP (Routing Information Protocol) - 路由信息协议

  2. OSPF (Open Shortest Path First) - 开放最短路径优先协议

而BGP（Border Gateway Protocol边界网关协议）是在自治系统外部的协议。

如图所示，画圈的属于一个自治系统，边界网关之间的连接属于BGP协议。

RIP(routing information protocal)

  • 采用距离-向量算法

  • 基于分布式的基于距离向量的路由选择协议，是因特网的协议标准，最大的优点是简单。

  • 维护从他自己到每一个目的网络的唯一最佳举例记录。

    • 距离：指的是跳数

    • 距离为1表示和路由器自己直接相连的网络

如图所示，rip对应的路由表如图所示，直接交付说明路由器和目的网络直接相连。

  • rip要求一条路由最多只能包含15个路由器，所以16不可达

rip路由表如何完善

  • 仅仅和相邻路由器交换信息

  • 路由器交换的信息是自己的路由表

  • 每30秒交换一次信息，然后更新信息。

    • 如果超过30分钟没有邻居的信息通告，则判定邻居没了，并更新自己的路由表。

  • 路由表刚开始工作的时候，只知道直连的网络（距离为1），接着每一个路由器（30秒一次）也只会和相邻的路由器交换信息并更新路由信息（只路由表中的所有信息）。

  • 经过若干次更新之后，所有路由器都会知道到达本自治系统的任何一个网络的最短距离和下一跳地址，即“收敛”。

距离向量算法

  1. 修改相邻路由器发来的RIP报文中所有表项

    1. 对相邻路由器X，修改此报文的所有项目，把下一跳的地址改为X，并把所有字段的距离+1。

    2. 例如R1就是这么做的

  • 对修改之后RIP报文中的每一个项目：

    1. R1路由表中若没有net3（不包含相同的目的网络），则把项目直接填入R1路由表

    2. R1路由表汇总包含net3（包含相同的目的网络）

      1. 若下一跳是X，则直接更新

      2. 若下一跳不是X，分两种情况（例如对于Net3 2 R3，更短，更新成新的路由R3；对于Net3 3 R3不是更短，不更新）

        1. 原来距离比走X更短则更新

        2. 若大于等于原来距离则不做更新

  • 如果180s还没有收到相邻路由表的信息更新路由表，则将X标记为不可达，即把距离记为16

  • 返回

例题1：

首先要修改R4发来的路由信息表，即距离加一，并把下一条地址首先全置为R4

然后再进行进一步判断，如果没有的信息直接更新，目的网络相同且下一跳地址相同以新的为准，不同则采用短的那个。

例题2：

本质上距离-向量算法就是求最短的距离路径。

RIP报文格式

RIP是应用层协议，使用UDP传送数据。

RIP特点

好消息传得快，坏消息（故障消息）传递的慢（慢收敛）

比如考虑以下这种情况：

  • R1到网1的距离为16，但是此时R1收到了R2的RIP报文，该报文中包含目的网络为网1的表项

  • 此时R1按照标准操作，发现目的网络相同，下一跳地址不同，于是选择距离较短的，也就是1,3,R2

  • 然后R2也会受到R1的RIP报文执行类似操作，距离不断+1

  • 直到距离>=16,才会标记为不可达

总结来说就是一句话：距离为16的很难传递，因为距离-向量算法决定网络相同，下一跳地址不同取最短，所以优先传递最短的。

OSPF(Open Shortest Path First)

Open指的是ospf协议是不是受某一个厂家控制，而是公开发表的。

Shortest path first指的是使用了dij算法。

opsf的最主要特征就是使用了分布式的链路状态协议。

OSPF特点

  • 和谁交换：类似广播形式，使用“洪泛法”，每一个路由器都通过端口向所有的路由器发送信息。

  • 交换的信息：是本路由器相邻的所有路由器的链路状态（与哪些路由器相邻、链路的代价如费用、时延、距离、带宽等）

    • 而RIP是相邻路由器和距离。

  • 交换的时间：只有当链路状态发生变化的时候才会使用之前说过的“洪泛法”向所有的路由器发送信息。

最后，所有的的路由器都可以建立一个链路状态数据库，即全网拓扑图。

链路状态路由算法

  1. 每个路由器发现它的邻居结点【通过HELLO问候分组】，并了解各个邻居结点的网路地址

  2. 设置到他的每个邻居的成本度量metric

  3. 构造【DD数据库描述分组】，向邻居结点发送自己的链路状态数据库（全网的拓扑结构图）中的所有链路状态项目的摘要信息

  4. 如果邻站分组中的摘要自己都有，则不做处理；如果没有或者是新的，则发送【LSR(Link State Request Packet)链路状态请求分组】请求自己没有的和比自己新的信息。

  5. 收到邻站的LSR分组之后，发送【LSU(Link State Update Packet)链路状态更新分组】进行更新。

  6. 更新完毕之后，邻站返回一个【LSAck(Link State Acknowledgment Packet)链路状态确认分组】进行确认。

当某个路由器的链路状态发生变化时：

  • 泛洪发送【LSU（链路状态更新）分组】进行全网更新。

  • 其他路由器更新完毕后，返回【LSAck（链路状态确认）分组】进行确认。

  • 使用Dijkstra根据自己的链路状态数据库构造到其他节点间的最短路径。

OSPF的区域

OSPF分组

opsf直接使用ip数据报进行传送，是网络层协议。rip使用UDP，是应用层协议。

BGP协议

BGP协议是外部网关协议的一个代表

  • 和谁交换：与其他AS（Autonomous System（自治系统））的邻站BGP 发言人的交换信息

  • 交换什么：交换的网络可达性的信息，即要到达某个网络所要经历的一系列AS

  • 多久交换：发生变化时更新有变化的部分

BGP交换信息的过程

边界路由器既要明确在AS内部应该如何走；到达其他AS应该如何走。

交换的信息是经过AS的路径。

BGP报文格式

BGP是应用层协议，使用TCP。

BGP特点

BGP-4的四种报文

这些都是AS之间的eBGP的会话格式，AS内部的会话格式只有iBGP。

  1. OPEN（打开）报文：用来和相邻的另一个BGP发言人进行连接，并认证发送方

  2. UPDATE（更新）报文：通告新路径或者撤销原路径

  3. KEEPALIVE（保活）报文：在无UPDATE时，周期性确定邻站的连接性；也作为OPEN的确认

  4. NOTIFICATION（通知）报文：报告先前报文的差错；也用于关闭TCP连接

BGP路由选择

  • 本地偏好值最高的路由。

  • 选择AS跳数最少的路由（因为他是eBGP）

  • 热土豆路由选择算法

    • 顾名思义，热土豆烫手要抛出去，所以传给最靠近NEXT-HOP（起点路由器）

  • BGP标识符的数值最小的路由

    • 这个BGP标识符位于BGP报文的首部，是最后的方法。

Bellman-Ford算法易错点

当一个路由器检测到相邻网络不可达，不能直接置为16！

在 RIP（Routing Information Protocol） 中，如果一个路由器发现某个网络不可达（比如直连链路失效或邻居通告跳数=16），它不会直接标记为 16，而是会执行以下操作：

  • 如果存在其他邻居通告的相同网络​，且跳数 <16​，路由器会选择最小的跳数+1​（RIP 的跳数计算规则）。

  • 只有当所有邻居都通告跳数=16​（或没有其他路径），才会最终将该网络标记为 16（不可达）​。

总结一下，就是min{16，R1到该网络的距离，R2到该网络的距离…}

详见课后习题25。

分层层次的路由协议

分为两个

  1. 内部网关协议 IGP

如RIP和OSPF

  • 外部网关协议 EGP

如BGP-4

两种BGP

分为AS内部的BGP——iBGP。

AS外部的BGP：

Open，Update，Keepalive，Notification。

首先Open建立TCP连接之后发送，互相识别，协商参数；然后收到Open之后路由器用Keepalive表示接受连接；用Update为核心，用于撤销之前通知的路由；Notification用于发送检测到的差错。

RIP的优缺点

  • 实现简单，开销小。

  • 好消息传播的快。

但是：

  • 坏消息传播的慢——“慢收敛”（收敛指的是所有的路由都获得了其他的路由状态信息）

  • 最大使用距离为15，仅支持小的自治系统，网络规模越大，开销越大。

静态路由与动态路由

静态路由由管理员手动配置的路由条目，不会自动适应网络变化。

动态路由的路由器通过路由协议（如OSPF、BGP）自动学习并更新路由表（有两种方法，距离-向量法和链路状态算法）。
