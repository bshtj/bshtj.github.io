为什么使用long[]而不是long？

  1. 高效位操作​：

    • long 是 64 位，可以一次性处理 64 个位的逻辑运算（如 AND、OR、XOR），比逐位操作更高效。

    • CPU 指令集对 long 的位运算有原生支持，速度极快。

  2. 内存紧凑​：

    • 用 long[] 存储可以最小化内存占用。例如：

      • BitSet(64) 只需 1 个 long（8 字节），而用 boolean[] 需要 64 字节（每个 boolean 占 1 字节）。

    • 对稀疏位集（大部分位为 0），BitSet 会动态压缩存储（如全 0 的 long 可能被忽略）。

  3. 动态扩容​：

    • words 数组会按需扩容。例如：

      • 初始 BitSet(16) 分配 long[1]。

      • 设置 bits.set(100) 后，words 扩容为 long[2]（因为 100 / 64 = 1，需要 words[0] 和 words[1]）。

这段代码展示了 Java 中 BitSet 类的多种实际用途，主要体现了以下几个核心作用：

1. 高效存储和操作布尔值集合

BitSet 用位（0/1）代替布尔数组，​极大节省内存​：

  • 示例​：containChars() 方法统计字符串中的唯一字符。

    • 每个字符的 ASCII 值作为索引，用 1 个 BitSet 代替 boolean[65536]。

    • 内存占用从 64KB 降至 8KB​（假设存储所有 ASCII 字符）。

2. 快速去重和排序

  • 示例​：sortArray() 方法对整数数组去重并排序。

    • 将数字直接作为索引设置到 BitSet 中（如 bitSet.set(423)）。

    • 自动去重（重复数字多次设置同一位置）。

    • 通过 nextSetBit() 遍历已设置的位，​天然有序输出​（升序）。

3. 数学算法优化（如素数筛）​

  • 示例​：computePrime() 实现埃拉托斯特尼筛法。

    • 用 BitSet 标记非素数，​比数组更省空间。

    • 位操作（set/clear）比数组赋值更快。

4. 二进制与字节数据转换

  • 示例​：bitSet2ByteArray() 和 byteArray2BitSet()。

    • 序列化​：将 BitSet 转换为紧凑的 byte[]（如网络传输）。

    • 反序列化​：从 byte[] 恢复 BitSet 状态。

5. 集合运算（交集、差集等）​

  • 示例​：simpleExample() 中的 andNot() 操作。

    • 支持逻辑运算（AND、OR、XOR），适合处理权限位掩码等场景
