不同形式的二分查找

发表于2025-05-13|更新于2025-05-13
|浏览量:

标准写法

二分查找让人疑惑的点就是什么时候while(left<right)什么时候while(left<=right)。

记住左闭右开区间为前者，因为不可能出现left = right的情况。左闭右闭为后者。

因为left == right的时候，在[left, right)是无效的空间，可以理解为[1,1)不行但是[1,1]可以.

关键点分析

  1. while (low < high) 而不是 while (low <= high)
    • 这种写法保证循环结束时 low == high，即最终返回的 low 就是目标位置。

    • 它不要求精确匹配，而是找到第一个 >= target 的位置（即插入位置）。

  2. if (f.get(mid) < target) 时 low = mid + 1
    • 如果 mid 处的值 < target，说明目标在右半部分，所以 low 直接跳到 mid + 1。

  3. else 时 high = mid
    • 如果 mid 处的值 >= target，说明目标可能在左半部分或 mid 就是第一个满足条件的点，所以 high 收缩到 mid（而不是 mid - 1），避免错过可能的解。

  4. 为什么可以省略 == 的情况？
    • 因为 f.get(mid) == target 时，high = mid 仍然会保留这个位置，继续向左收缩，直到找到最左边的 >= target 的位置。

关键点分析

  1. mid 计算方式改为 (low + high + 1) / 2
    • 因为 low = mid 可能导致死循环（如 low = 1, high = 2 时，mid = 1，如果 low = mid，则 low 不会变化）。

    • 向上取整可以避免死循环。

  2. if (f.get(mid) > target) 时 high = mid - 1
    • 如果 mid 处的值 > target，说明目标在左半部分，所以 high 跳到 mid - 1。

  3. else 时 low = mid
    • 如果 mid 处的值 <= target，说明目标可能在右半部分或 mid 就是最后一个满足条件的点，所以 low 移动到 mid（而不是 mid + 1），避免错过解。

  4. 适用场景
    • 这种写法适用于寻找最后一个 <= target 的位置（即右边界）。

总结

写法 适用场景 特点
while (low < high) + high = mid 找第一个 >= target 的位置（左边界） 适用于 lower_bound
while (low < high) + low = mid 找最后一个 <= target 的位置（右边界） 适用于 upper_bound
while (low <= high) + 三种情况 标准二分查找（精确匹配） 适用于查找是否存在

如何选择？
• 如果问题要求插入位置（如最长递增子序列），用 while (low < high) + high = mid（原写法）。

• 如果问题要求精确匹配，用 while (low <= high) + 三种情况。

• 如果问题要求右边界，用 while (low < high) + low = mid（但要注意 mid 计算方式）。
