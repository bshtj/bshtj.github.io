Java Comparator 接口方法详解

Comparator 是 Java 中用于定义对象比较规则的接口，位于 java.util 包中。以下是 Comparator 接口中的所有方法及其说明：

1. 核心比较方法

int compare(T o1, T o2)

  • 功能：比较两个对象
  • 参数：o1 - 第一个对象，o2 - 第二个对象
  • 返回：
    • 负整数：o1 < o2
    • 零：o1 == o2
    • 正整数：o1 > o2

2. 默认方法（Java 8+）

Comparator<T> reversed()

  • 功能：返回一个逆序比较器
  • 示例：

Comparator<T> thenComparing(Comparator<? super T> other)

  • 功能：当第一个比较器比较结果为相等时，使用另一个比较器继续比较
  • 示例：

default <U> Comparator<T> thenComparing(Function<? super T, ? extends U> keyExtractor, Comparator<? super U> keyComparator)

  • 功能：先按当前比较器比较，相等时再按提取的键和指定的比较器比较
  • 示例：

default <U extends Comparable<? super U>> Comparator<T> thenComparing(Function<? super T, ? extends U> keyExtractor)

  • 功能：先按当前比较器比较，相等时再按提取的键的自然顺序比较
  • 示例：

default Comparator<T> thenComparingInt(ToIntFunction<? super T> keyExtractor)

  • 功能：先按当前比较器比较，相等时再按int类型键比较
  • 示例：

default Comparator<T> thenComparingLong(ToLongFunction<? super T> keyExtractor)

  • 功能：先按当前比较器比较，相等时再按long类型键比较
  • 示例：

default Comparator<T> thenComparingDouble(ToDoubleFunction<? super T> keyExtractor)

  • 功能：先按当前比较器比较，相等时再按double类型键比较
  • 示例：

3. 静态方法（Java 8+）

static <T extends Comparable<? super T>> Comparator<T> naturalOrder()

  • 功能：返回自然顺序比较器
  • 示例：

static <T extends Comparable<? super T>> Comparator<T> reverseOrder()

  • 功能：返回逆序自然顺序比较器
  • 示例：

static <T, U> Comparator<T> comparing(Function<? super T, ? extends U> keyExtractor, Comparator<? super U> keyComparator)

  • 功能：根据提取的键和指定的比较器创建比较器
  • 示例：

static <T, U extends Comparable<? super U>> Comparator<T> comparing(Function<? super T, ? extends U> keyExtractor)

  • 功能：根据提取的键的自然顺序创建比较器
  • 示例：

static <T> Comparator<T> comparingInt(ToIntFunction<? super T> keyExtractor)

  • 功能：根据int类型键创建比较器
  • 示例：

static <T> Comparator<T> comparingLong(ToLongFunction<? super T> keyExtractor)

  • 功能：根据long类型键创建比较器
  • 示例：

static <T> Comparator<T> comparingDouble(ToDoubleFunction<? super T> keyExtractor)

  • 功能：根据double类型键创建比较器
  • 示例：

static <T> Comparator<T> nullsFirst(Comparator<? super T> comparator)

  • 功能：将null视为小于非null值，非null值使用指定比较器比较
  • 示例：

static <T> Comparator<T> nullsLast(Comparator<? super T> comparator)

  • 功能：将null视为大于非null值，非null值使用指定比较器比较

如何使用装饰器模式实现链式调用

装饰器模式非常适合实现链式调用，因为它允许动态地添加功能到对象中。下面我将展示如何用装饰器模式实现一个支持链式调用的 Comparator。

2. 装饰器实现

装饰器模式是一种 结构型设计模式，它允许 动态地扩展对象的功能，而 不改变其原始结构。它的核心特点是：

  • 装饰器（Decorator） 持有 被装饰对象（Component） 的引用，并实现相同的接口。
  • 装饰器可以在 调用被装饰对象的方法前后 添加额外的逻辑。
  • 可以 嵌套多个装饰器，形成链式增强。

ThenComparingComparator 如何体现装饰器模式？

(1) 它实现了 MyComparator 接口，并持有另一个 MyComparator

  • first 是被装饰的原始比较器（如 PropertyComparator）。
  • second 是新增的比较逻辑（如另一个 PropertyComparator 或 ThenComparingComparator）。
  • **ThenComparingComparator 本身也是一个 MyComparator**，所以它可以 递归嵌套，形成链式调用。

(2) 它增强（装饰）了 compare 方法

  • **先调用 first.compare()**（原始比较逻辑）。
  • **如果 first 返回 0（相等），再调用 second.compare()**（新增比较逻辑）。
  • 这样，ThenComparingComparator 在不修改 first 和 second 的情况下，扩展了比较逻辑。

(3) 可以无限嵌套，形成链式装饰

  • 每次调用 thenComparing，都会 **创建一个新的 ThenComparingComparator**，并 **把当前的比较器作为 first，新的比较器作为 second**。
  • 最终形成一个 嵌套的装饰链
  • 这样，每层装饰器都能增强比较逻辑，而 不修改原有的比较器。

装饰器模式的关键点

  1. 包装对象：装饰器类持有被装饰对象的引用
  2. 保持接口一致：装饰器与被装饰对象实现相同接口
  3. 动态扩展：可以在运行时组合多个装饰器
  4. 透明性：客户端无需知道是否使用了装饰器

这种实现方式与 JDK 中的 Comparator 实现原理非常相似，只是简化了一些细节。通过装饰器模式，我们可以灵活地组合多个比较逻辑，形成链式调用。
