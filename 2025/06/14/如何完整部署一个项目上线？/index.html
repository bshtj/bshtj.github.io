如何完整部署一个项目上线？

发表于2025-06-14|更新于2025-06-14
|浏览量:

注：本文使用的是vue前端以及springboot后端。

部署后端

Mvn Package

使用指令mvn package

为防止报错，可以禁用test测试。

将生成的jar包命名为自己的名称，如acss-backend。然后放入服务器根目录。

配置Dockerfile

命名文件为Dockerfile，放在服务器根目录。

构建docker镜像并运行容器

使用Dockerfile构建镜像。

  • -t acss-backend 指定镜像名称为 acss-backend

  • 末尾的 . 表示 Dockerfile 位于当前目录

构建完后运行容器

  • -d 后台运行

  • -p 8080:8080 将容器内8080端口映射到主机8080端口（根据你的jar实际端口调整）

    • 第一个8080是主机的端口，第二个是容器的端口

  • --name acss-container 为容器命名

报错处理

1.404

首先检查nginx的 proxy_pass 配置（结尾是否加/），给出以下结论。

前端请求 URL http://frontend/api/user/login proxy_pass 配置 实际代理到后端的 URL 替换逻辑 适用场景
location /api/ { proxy_pass http://backend:8080/api/; } http://backend:8080/api/（带 /） http://backend:8080/api/user/login 等量替换： /api/ → /api/（完全保留前缀） 后端接口本身包含 /api 前缀
location /api/ { proxy_pass http://backend:8080/; } http://backend:8080/（仅根 /） http://backend:8080/user/login 根路径替换： /api/ → /（完全丢弃前缀） 后端接口无前缀（如直接 /user/login）
location /api/ { proxy_pass http://backend:8080/newapi/; } http://backend:8080/newapi/ http://backend:8080/newapi/user/login 自定义替换： /api/ → /newapi/（修改前缀） 后端接口需要统一前缀但名称不同
location /api/ { proxy_pass http://backend:8080/api; } http://backend:8080/api（无 /） http://backend:8080/apiuser/login ❌ 错误拼接： location 的 /api + 请求的 /user/login（导致路径断裂） 永远不要这样写！
