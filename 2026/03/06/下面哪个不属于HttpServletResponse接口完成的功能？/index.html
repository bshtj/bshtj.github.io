下面哪个不属于HttpServletResponse接口完成的功能？

发表于2026-03-06|更新于2026-03-06
|浏览量:

一、答案：C

读取路径信息不属于HttpServletResponse的功能，该功能由HttpServletRequest（请求对象）完成；而HttpServletResponse核心负责向客户端返回响应数据、设置响应相关的属性（头标、Cookie 等）。

二、逐选项解析（附用法 + 代码示例）

先明确核心定位：

  • HttpServletRequest：封装客户端向服务器发送的请求，用于读取请求相关数据（路径、参数、请求头、Cookie 等）；
  • HttpServletResponse：封装服务器向客户端返回的响应，用于写入 / 设置响应相关数据（响应头、Cookie、返回内容等）。

选项 A：设置 HTTP 头标（属于 HttpServletResponse）

功能：向响应中添加 / 修改 HTTP 头信息（如设置响应编码、缓存策略、内容类型、重定向等）。

常用方法：

  • setHeader(String name, String value)：设置自定义 / 通用响应头；
  • setContentType(String type)：快捷设置Content-Type头（如text/html;charset=UTF-8）；
  • setCharacterEncoding(String charset)：设置响应体编码；
  • addHeader(String name, String value)：添加同名多值的响应头（区别于setHeader覆盖原有值）。

功能：服务器通过response向客户端写入 Cookie（客户端会保存该 Cookie，后续请求会携带）。

常用方法：addCookie(Cookie cookie)。

选项 C：读取路径信息（不属于 HttpServletResponse，属于 HttpServletRequest）

功能：读取客户端请求的 URL 路径相关信息（如请求的上下文路径、Servlet 路径、请求 URI 等），是HttpServletRequest的核心能力。

常用方法：

  • getContextPath()：获取当前 Web 应用的上下文路径（如/mywebapp）；
  • getServletPath()：获取请求匹配的 Servlet 路径（如/hello）；
  • getRequestURI()：获取完整请求 URI（如/mywebapp/hello）；
  • getPathInfo()：获取 Servlet 路径后的额外路径（如请求/mywebapp/hello/user/123，则返回/user/123）。

选项 D：输出返回数据（属于 HttpServletResponse）

功能：向客户端输出响应体数据（如 HTML、JSON、文本等），是response最核心的功能之一。

常用方法：

  • getWriter()：获取字符输出流（用于输出文本数据，如 HTML、JSON）；
  • getOutputStream()：获取字节输出流（用于输出二进制数据，如图片、文件）；
  • flushBuffer()：强制刷新缓冲区，将数据立即发送给客户端。

三、总结

  1. 核心分工：HttpServletRequest负责 “读” 请求数据（路径、参数、Cookie 等），HttpServletResponse负责 “写” 响应数据（头标、Cookie、返回内容等）；
  2. 关键区分：“读取路径信息” 是request的功能，而response仅负责设置响应相关属性、输出响应内容；
  3. 高频用法：
    • response：设置Content-Type、添加 Cookie、通过getWriter()输出文本 / JSON 是日常开发最常用的场景；
    • request：读取路径、请求参数、客户端 Cookie 是配套的核心操作。
